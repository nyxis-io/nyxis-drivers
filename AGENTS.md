# AGENTS.md

Guidance for maintainers, contributors, and automated assistants working in this repository.

## Overview

`nyxis-drivers` contains the MIT-licensed SDKs for reading, writing, prefetching, and benchmarking Nyxis `.nxb` payloads. The core spec, Rust compiler, fixture generator, conformance vectors, demos, benchmark harnesses, and MCP server live in the sibling `../nyxis` repo.

## Layout

| Directory | Language or surface |
| --- | --- |
| `c/` | C99 reader/writer shared by C extensions |
| `go/` | Go 1.26+ |
| `py/` | Python 3.8+ pure driver plus optional C extension |
| `js/` | JavaScript for Node/browser, ESM, WASM helpers |
| `ruby/` | Ruby pure driver plus C extension |
| `php/` | PHP pure driver plus C extension |
| `kotlin/` | Kotlin/JVM |
| `csharp/` | C# / .NET |
| `swift/` | Swift |
| `devtools-extension/` | Browser DevTools panel for `.nxb` responses |

Each language directory has its own `README.md` with install commands, API examples, and language-specific notes.

## Code Navigation

Use `docs/index/_root.md` as the LOI navigation index for task-oriented lookup. It routes driver tasks such as prefetch, C extensions, DevTools, and columnar layouts to focused room files.

Wire-format rules live in `../nyxis/SPEC.md`. Driver contribution conventions live in `CONTRIBUTING.md`.

## Setup

The `../nyxis` sibling repo must be present before regenerating fixtures or running cross-repo tests. The Makefile defaults `CORE` to `../nyxis`, or to `./nyxis` when CI checks out the core repo inside this repo.

```bash
make fixtures
```

Fixtures are gitignored. Fresh clones need `make fixtures` before most driver tests.

Install hooks once per checkout:

```bash
make install-git-hooks
```

The pre-commit hook runs `make lint`. Bypass a single commit with `SKIP_HOOKS=1 git commit ...`.

## Build, Test, Lint

Run all drivers:

```bash
make test
make lint
```

The aggregate `make fix` target currently references `fix-c`, but C has no real auto-fix recipe. Prefer the per-language fix targets below until that Makefile gap is closed.

Run a single driver while iterating:

```bash
make test-go      make lint-go      make fix-go
make test-js      make lint-js      make fix-js
make test-py      make lint-py      make fix-py
make test-c       make lint-c       # no C auto-fix
make test-ruby    make lint-ruby    make fix-ruby
make test-php     make lint-php     make fix-php
make test-kotlin  make lint-kotlin  # no fix-kotlin
make test-csharp  make lint-csharp  make fix-csharp
make test-swift   make lint-swift   make fix-swift
```

CI extras build C extensions before re-running language tests:

```bash
make test-py-ci
make test-ruby-ci
make test-php-ci
```

From the multi-repo parent directory:

```bash
make -C nyxis fixtures
make -C nyxis-drivers test
```

## Language Notes

- Go uses `gofmt` and `go vet ./...`; the module is `github.com/nyxis-io/nyxis-drivers/go` and requires Go 1.26+.
- Python uses `ruff check --select E,W,F --ignore E501,E701,E702`; tests receive the fixture path as their first argument.
- JavaScript is ESM and uses ESLint with `--max-warnings 0` on core files (`nxs.js`, `nxs_writer.js`, `wasm.js`, `test.js`, `test_wasm.js`).
- Ruby uses RuboCop with `ruby/.rubocop.yml`.
- PHP uses PHPStan level 5. `fix-php` aliases `lint-php`.
- C uses `cppcheck`; there is no formatter target.
- Kotlin uses Gradle `ktlintCheck`; the Makefile sets `JAVA_HOME=/opt/homebrew/opt/openjdk@21` by default.
- C# uses `DOTNET_FRAMEWORK=net10.0` and the custom `NxsTargetFramework` property.
- Swift uses SwiftLint with `--strict` and `.swiftlint-cache/`.
- DevTools changes may need `bash devtools-extension/sync-lib.sh` after updating shared JS decode code.

## Conformance

Conformance vectors live in `../nyxis/conformance/` and are generated by the core repo. Run the full suite from the core repo:

```bash
make -C ../nyxis conformance
make -C ../nyxis conformance-prefetch PREFETCH=1
```

For core format changes, update vectors in `../nyxis` first, then adapt drivers.

## Conventions

- TypeManifest sigils must match the core spec and `nxs::consts` in `../nyxis/rust`.
- Match existing per-language API conventions, such as `NewWriter`, `Reader`, and column reducer naming.
- New writers should have round-trip coverage against Rust-generated vectors where possible.
- Shared C changes can affect Python, Ruby, and PHP extensions; run the relevant `test-*-ci` targets when touching shared C code.
- Keep PRs focused to one language driver or one cross-cutting concern when possible.

## CI

Driver workflows check out `nyxis` to build fixtures and conformance vectors. Both `nyxis` and `nyxis-drivers` need the `NYXIS_CI_AUTOMATION_TOKEN` secret for cross-repo jobs; see `.github/workflows/README.md`.
